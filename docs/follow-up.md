# 前端界面后续优化任务

## 高优先级

- [x] 将底部浮动条改为真实连接状态栏。
  - 目标：替换当前开发说明文案，显示当前可用通道和下一步操作。
  - 建议状态：`云端未登录`、`云端已登录`、`蓝牙未连接`、`蓝牙已认证`、`双通道可用`。
  - 验收：用户不用打开日志也能判断当前是否可控车，以及应该点“连接”还是直接控车。

- [x] 让控车反馈圆环只在执行态动画。
  - 目标：避免 Idle/Ready 状态下持续转动造成“正在执行命令”的误解。
  - 建议状态：`Idle`、`Ready` 静止；`TX`、`Hold`、`Timeout`、`Fail` 才使用动画或强调色。
  - 验收：未发送命令时反馈区保持稳定，发送中才出现明显运动反馈。

- [x] 为危险动作长按加入进度反馈。
  - 目标：设防、断电长按 1 秒时，用户能看到触发进度。
  - 建议实现：按钮内进度条、环形进度、或按住时文字从“长按确认”变为“继续按住”。
  - 验收：松手取消和长按触发在视觉上明确可区分。

- [x] 统一顶部连接状态表达。
  - 目标：减少云端 `cloud-state` 和 BLE `conn-state` 分裂导致的理解成本。
  - 建议实现：顶部 pill 根据当前控制通道显示统一状态，Hero 内保留链路明细。
  - 验收：用户能一眼看出当前是云控、蓝牙控车，还是离线不可控车。

## 中优先级

- [x] 优化云端/BLE 切换后的抽屉展开体验。
  - 目标：切换模式后连接面板展开更自然。
  - 建议实现：展开动画、自动滚动到连接区域、或在按钮旁显示“已展开连接设置”提示。
  - 验收：切换模式后用户能立即看到对应登录/配对入口。

- [x] 压缩小屏首屏 Hero 高度。
  - 目标：让 360px 宽左右设备上能更早看到常用控车按钮。
  - 建议实现：小屏缩短车图高度、降低 Hero padding、压缩状态 chip 高度。
  - 验收：小屏首屏至少露出“常用控车”标题和部分按钮。

- [x] 优化无数据状态说明。
  - 目标：电池、电压、车辆状态为 `--` 时解释数据来源。
  - 建议文案：`登录或连接后更新`、`等待车辆回传`。
  - 验收：空状态不再像加载失败或页面坏掉。

- [x] 调整控车按钮文案。
  - 目标：降低新用户理解成本。
  - 建议文案：`解防解锁`、`一键上电`、`寻车鸣笛`、`安全设防`、`切断电门`、`打开坐垫`。
  - 验收：不依赖单字图标也能明确按钮含义。

- [x] 降低工程调试入口存在感。
  - 目标：普通用户首屏聚焦连接和控车。
  - 建议实现：底部浮动条只保留“连接”，调试入口移动到连接抽屉底部或二级入口。
  - 验收：默认界面不像工程调试工具，调试能力仍能找到。

- [x] 拆分验证码登录态和未登录态布局。
  - 目标：避免手机窄屏下输入框和多个按钮挤在一行。
  - 建议实现：未登录时显示手机号/验证码/登录；登录后显示已登录状态和退出按钮。
  - 验收：320-370px 宽度下表单不拥挤、不换行混乱。

## 低优先级

- [x] 统一中英文小标题风格。
  - 目标：减少原型感。
  - 建议：将 `Primary actions`、`Cloud link`、`Engineering mode` 改成中文，或使用更克制的中英混排。
  - 验收：界面语言风格统一。

- [x] 增加关键异常空状态卡片。
  - 目标：让验证码失败、无车辆、浏览器不支持 Web Bluetooth、非 HTTPS 等场景不只依赖日志。
  - 建议实现：连接抽屉内显示可操作提示和恢复步骤。
  - 验收：用户无需打开工程日志即可理解问题。

- [x] 减少非必要入场动画。
  - 目标：让控车工具更稳定、少干扰。
  - 建议保留首屏核心入场动画，减少按钮和卡片重复动画。
  - 验收：页面加载仍有质感，但控车区域不显得跳动。

- [x] 继续跟进 `llms.txt` 审计兼容性。
  - 目标：如果需要追求 Agentic Browsing 分数，再定位 Lighthouse 对内容格式或路径的具体要求。
  - 验收：常规 Lighthouse 分数不受影响，必要时 Agentic Browsing 单项通过。
